## 검증 목적

이 프로그램은 사무실의 냉간성형강 구조설계에 사용될 예정이므로, 다음 3가지 관점에서 모든 설계 코드를 빠짐없이 검증한다:

1. **AISI S100-16 기준 적합성** — US 단위계로 작성된 설계 공식이 AISI 기준을 정확히 따르는가?

3. **단위 혼용 문제** — 코드 내에서 단위계가 혼재되어 계산 오류를 유발하는 부분이 있는가?

### 1.2 전체좌굴 — `global_buckling.py`

#### 1.2.1 column_global_strength (§E2) — [OK]
- **λc = √(Fy/Fcre)** ✓
- **λc ≤ 1.5 → Fn = 0.658^(λc²) × Fy** (Eq. E2-2) ✓
- **λc > 1.5 → Fn = 0.877/λc² × Fy** (Eq. E2-3) ✓

#### 1.2.2 flexural_buckling_stress (§E2.1) — [OK]
- **σex = π²E / (KL/r)²** ✓

#### 1.2.3 torsional_buckling_stress (§E2.1) — [OK]

#### 1.2.4 flexural_torsional_stress (§E2.2) — [OK]
- **β = 1 - (xo/ro)²** ✓
- **Fcre = 1/(2β) × [(σex+σt) - √((σex+σt)² - 4β×σex×σt)]** (Eq. E2.2-1) ✓

#### 1.2.5 compute_column_Fcre (§E2) — [OK]
- 이중대칭: min(σex, σey, σt) ✓
- 단축대칭: 휨-비틀림좌굴 적용 ✓

#### 1.2.6 beam_global_strength (§F2) — [OK]
- **Fcre ≥ 2.78Fy → Fn = Fy** (yielding) ✓
- **0.56Fy < Fcre < 2.78Fy → Fn = (10/9)Fy(1 - 10Fy/36Fcre)** (inelastic LTB) ✓
- **Fcre ≤ 0.56Fy → Fn = Fcre** (elastic LTB) ✓
- **Mne = Sf × Fn** ✓

#### 1.2.7 beam_global_strength — Inelastic Reserve (§F2.4.2) — [WARN]
- **파일:** `global_buckling.py:163-168`
- **코드:**
  ```python
  ratio = math.sqrt(My / Mcre)
  Mne_ir = Mp - (Mp - My) * (ratio - 0.23) / 0.37
  ```
- **AISI F2.4.2-1:** Mne = Mp - (Mp - My) × [(Cyp - 1)/(Cyp - 0.23/0.37)] 형태
- **상태:** 공식의 정확한 형태를 AISI S100-16 원문과 대조 확인 필요. 코드의 `(ratio - 0.23) / 0.37` 표현이 Cyp 기반 공식과 동일한지 확인 요망.
- **위험도:** 중 — Inelastic Reserve는 선택적 기능(기본 비활성)이므로 일반 설계에는 영향 없음.

#### 1.2.8 compute_beam_Fcre (§F2.1.1) — [OK]
- **C-section:** Fcre = Cb × ro × A / Sf × √(σey × σt) (Eq. F2.1.1-1) ✓
- **Z-section (§F2.1.3):** Fcre = Cb × ro × A / (2×Sf) × √(σey × σt) ✓ (분모 2 적용)

#### 1.3.2 web_crippling (§G5) — [ISSUE-01] fastened/unfastened 계수 동일 오류
- **파일:** `shear.py:50-62`
- **문제:** `_WC_COEFFS` 딕셔너리에서 fastened와 unfastened의 계수(C, Cr, CN, Ch)가 모든 하중 케이스(EOF, IOF, ETF, ITF)에 대해 동일하게 입력되어 있음.
- **코드:**
  ```python
  ('EOF', 'fastened'): (4.0, 0.14, 0.35, 0.02),
  ('EOF', 'unfastened'): (4.0, 0.14, 0.35, 0.02),  # 동일!
  ```
- **AISI Table G5-3 (C/Z 단일 비보강 웹):**
  | Case | Condition | C | Cr | CN | Ch |
  |------|-----------|---|----|----|-----|
  | EOF | Unfastened | 4.0 | 0.14 | 0.35 | 0.02 |
  | EOF | Fastened | 4.0 | 0.40 | 0.60 | 0.03 |
  | IOF | Unfastened | 13.0 | 0.23 | 0.14 | 0.01 |
  | IOF | Fastened | 13.0 | 0.32 | 0.05 | 0.04 |
  | ETF | Unfastened | 6.9 | 0.06 | 0.16 | 0.05 |
  | ETF | Fastened | 6.9 | 0.14 | 0.07 | 0.04 |
  | ITF | Unfastened | 13.0 | 0.32 | 0.05 | 0.04 |
  | ITF | Fastened | 17.0 | 0.30 | 0.05 | 0.04 |
- **영향:** fastened 조건에서 웹크리플링 강도가 과소평가될 수 있음 (보수적이지만 비경제적).
  unfastened 값이 fastened보다 대체로 낮으므로 안전측이기는 하나, IOF/ITF fastened에서 C=13이 아닌 17이어야 하는 등 정확도가 떨어짐.
- **위험도:** 중 — 웹크리플링 검토 결과에 직접 영향.

### 1.5 조합하중 상관식 — `interaction.py`

#### 1.5.1 combined_axial_bending (§H1.2) — [ISSUE-02] P/Pa ≤ 0.15 분기 누락
- **파일:** `interaction.py:6-30`
- **코드:** `total = P/Pa + Mx/Max + My/May` (단일 공식만 사용)
- **AISI S100-16 §H1.2:**
  - **P/Pa > 0.15** → Eq. H1.2-1 (모멘트 증폭 포함): P/Pa + CmxMx/[Max(1-P/PEx)] + CmyMy/[May(1-P/PEy)] ≤ 1.0
  - **P/Pa ≤ 0.15** → Eq. H1.2-2 (단순 선형): P/(2Pa) + Mx/Max + My/May ≤ 1.0
- **현재 코드:** `_design_combined()`에서 모멘트 증폭(Cm/(1-P/PE))을 먼저 적용한 후 `combined_axial_bending()`에 증폭된 모멘트를 전달. 하지만:
  1. H1.2-2의 **P/(2Pa)** 형태가 구현되지 않음 (항상 P/Pa 사용)
  2. P/Pa ≤ 0.15일 때도 모멘트 증폭이 적용됨 (과도하게 보수적)
- **영향:** 항상 보수적(안전측)이므로 위험하지는 않으나, 축력이 작은 경우(퍼린 등) 불필요하게 보수적인 결과를 줌.
- **위험도:** 중 — 경제성에 영향. 안전성에는 문제 없음(보수적).

### 4.14 [RISK-10] screw_connection의 t2/t1 보간 구간 누락
- **파일:** `connections.py:163-179`
- **코드:**
  ```python
  if t2 / t1 <= 1.0:          # Case 1: t2/t1 ≤ 1.0
      alpha = 4.2 * (t2/d)**0.5 if t2/d <= 1.0 else 2.7
      Pns = alpha * t2 * d * Fu * n
  elif t2 / t1 >= 2.5:        # Case 3: t2/t1 ≥ 2.5
      Pns = 2.7 * t1 * d * Fu * n
  else:                        # Case 2: 1.0 < t2/t1 < 2.5 (보간)
      Pns1 = 4.2 * (t2/d)**0.5 * t2 * d * Fu * n
      Pns2 = 2.7 * t1 * d * Fu * n
      Pns = min(Pns1, Pns2)
  ```
- **AISI S100-16 §J4.3.1:**
  1.0 < t2/t1 < 2.5 구간에서는 Case 1과 Case 3의 **선형 보간**(linear interpolation)을 요구.
  코드는 `min(Pns1, Pns2)`(두 값의 최소)를 사용하는데, 이는 정확한 선형 보간이 아님.
  `min`은 보수적 결과를 주므로 안전측이나, 기준과 정확히 일치하지 않으며 지나치게 보수적일 수 있음.

## Part 5: 다각도 오류 가능성 분석 — 사용 시나리오, 상태 정합성, 부호 규약, 수치 안정성

### 5.1 [ISSUE-13] DSM 응력 상태 불일치 — 가장 위험한 사용 시나리오

- **파일:** `StcfsdPanel.ts:727-748`
- **시나리오:**
  1. 사용자가 전처리에서 "pure_bending" 응력 설정 → FSM 해석 실행
  2. 해석 결과(`curve`)는 **휨 응력 기반** 좌굴 곡선
  3. 설계 탭에서 "compression" 부재 설계 선택
  4. 코드가 **동일한 curve**에서 `dsmP`(압축)와 `dsmM`(휨) 모두 추출:
     ```typescript
     const dsmP = await this._pythonBridge.call('dsm', {
         curve: this._lastAnalysisResult.curve,  // 휨 해석 결과!
         fy: aFy, load_type: 'P',                // 압축용 추출 요청
     });
     ```
  5. DSM 추출 시 `Pcrl = LF_local × Py` — 여기서 LF는 **휨 응력 기반** load factor
  6. 휨 응력으로 구한 LF를 Py에 곱하면 **압축 Pcrl이 아닌 전혀 다른 값** 산출

- **dsm.py 코드 확인:** `extract_dsm_values(curve, node, elem, fy, load_type='P')` 에서:
  ```python
  if load_type == 'P':
      P_ref = Py          # ← Py = Ag × Fy
  elif load_type == 'Mxx':
      P_ref = My_xx       # ← My = Sf × Fy
  # ...
  Pcrl = minima[0]['load_factor'] * P_ref
  ```
  curve의 load_factor는 해석 시 **적용된 응력의 배수**이므로, 응력 상태와 load_type이 일치하지 않으면 의미 없는 값이 됨.

- **영향:** 압축 부재 설계 시 Pcrl이 크게 틀려서 **국부좌굴 강도가 과대 또는 과소** 평가됨.
  과대 평가 시 비안전측.
- **위험도:** **매우 높음** — 일반적인 사용 시나리오에서 쉽게 발생.
- **해결:** 해석 시 사용한 응력 상태를 기록하고, 설계 load_type과 불일치 시 경고/차단.

### 5.2 [ISSUE-14] 해석-설계 간 단면 불일치 (상태 동기화 없음)

- **파일:** `StcfsdPanel.ts:723-756`
- **시나리오:**

### 5.10 [RISK-16] NaN 전파 위험

- **잠재 경로:**
  1. `sqrt(Pne / Pcrl)` — Pcrl < 0이면 음수/음수 = 양수이므로 OK. Pne < 0이면 NaN.
  2. `math.sqrt(rx**2 + ry**2 + xo**2)` — 항상 양수이므로 OK.
  3. `0.658 ** (lam_c ** 2)` — lam_c가 매우 크면 0에 수렴 (OK), lam_c가 NaN이면 전파.
  4. `Ixx / cz` — cz가 0이면 ZeroDivision. `cz = max(...) or 1.0`으로 방어하나, max가 정확히 0이면 통과.
- **실제 위험:** Pcrl이 음수가 될 수 있는가? → FSM 해석에서 load_factor가 음수일 수 있음 (인장이 우세한 응력 분포). 이 경우 `Pcrl < 0` → `λl = sqrt(Pne / negative)` = NaN → 이후 모든 계산에 NaN 전파.
- **방어 코드:** `if Pcrl <= 0: Pnl = Pne` — 이 방어가 있으므로 Pcrl ≤ 0은 처리됨 ✓
  그러나 `dsm.py:68`에서 `if row[1] > 0: points.append(...)` — 음수 LF는 필터링됨 ✓
- **위험도:** 낮 — 대부분의 NaN 경로가 방어되어 있으나, 예상치 못한 입력 조합에서 가능.
