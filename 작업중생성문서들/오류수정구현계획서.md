# 오류 수정 구현계획서

기준 문서: `코드검증프로세스.md`

목표는 `F-001`부터 현재 기록된 항목까지를 구현 관점에서 묶어, 수정 순서와 코드 영향 범위를 명확히 한 뒤 실제 코드 수정과 회귀 검증으로 연결하는 것이다.

## 1. 구현 원칙

- 내부 계산 단위는 기존처럼 US 단위를 유지한다.
- UI, 보고서, 검증 탭, MCP 설명은 실제 구현 수준과 상태 의존성을 정확히 반영한다.
- 설계/해석 상태는 “현재 모델과 현재 응력 상태”에 종속되므로, 상태가 바뀌면 이전 해석 결과를 재사용하지 않는다.
- 다경간/랩/부모멘트는 “대표 1개”로 축약하지 않고, 최소한 지배 구간을 보존해서 설계·검증·보고서에 전달한다.
- AISI 조항 구현이 축약식인 경우, 숨기지 않고 입력 제한 또는 경고로 명시한다.
