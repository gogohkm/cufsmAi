# CUFSM MATLAB → VS Code Extension 컨버전 전략

## 1. 결론: 컨버전 가능 여부

**컨버전은 충분히 가능하다.** VS Code Extension + WebView 구조로 전환한다.

| 영역 | MATLAB 코드량 | 전환 대상 | 난이도 |
|------|-------------|-----------|--------|
| 해석 엔진 (analysis/) | ~1,650 LOC | Python (NumPy/SciPy) | 낮음 |
| 모드 분류 (cFSM/) | ~800 LOC | Python (SciPy) | 중간 |
| 시각화 (plotters/) | ~3,700 LOC | WebView (Canvas/SVG/Babylon.js) | 중간 |
| 유틸리티 (helpers/) | ~2,000 LOC | Python | 낮음 |
| GUI (interface/) | ~11,300 LOC | WebView (HTML/CSS/JS) **재설계** | 높음 |

## 2. 전체 아키텍처

VS Code Extension을 호스트로, Python을 해석 백엔드로, WebView를 GUI로 사용하는 3계층 구조.

```
┌─────────────────────────────────────────────────────────────┐
│  VS Code Extension (TypeScript)                             │
│  ├── extension.ts         : 진입점, 커맨드 등록             │
│  ├── CufsmPanel.ts        : WebView 패널 생성/관리          │
│  ├── ProjectExplorer.ts   : 트리뷰 (프로젝트 탐색기)        │
│  └── PythonBridge.ts      : Python 프로세스 통신 (JSON-RPC) │
│         │                          │                        │
│    postMessage()              child_process.spawn()         │
│         │                          │                        │
│         ▼                          ▼                        │
│  ┌─────────────┐          ┌──────────────────┐              │
│  │  WebView     │          │  Python Engine   │              │
│  │  (HTML/JS)   │          │  (NumPy/SciPy)   │              │
│  │             │          │                  │              │
│  │  입력 폼     │          │  fsm_solver.py   │              │
│  │  데이터 테이블│  ◄─JSON─► │  cfsm/           │              │
│  │  차트 (Canvas)│          │  plastic/        │              │
│  │  3D (Babylon) │          │  vibration/      │              │
│  └─────────────┘          └──────────────────┘              │
└─────────────────────────────────────────────────────────────┘
```

### 왜 이 구조인가

| 선택 | 이유 |
|------|------|
| **Python 백엔드** (TypeScript로 해석 엔진을 짜지 않음) | `scipy.sparse.linalg.eigsh` — 희소 일반화 고유치 문제 풀이를 JS/TS에서 신뢰성 있게 구현하기 어려움. `null_space`, `orth`, `cholesky` 등 수치선형대수 생태계가 Python/SciPy에서 성숙함 |
| **WebView GUI** (PyQt 아님) | VS Code Extension으로 배포, IDE 내 통합, 기존 StSolar/EnergyPlus/stgen 프로젝트와 동일한 패턴 |
| **Babylon.js** (Three.js 아님) | stgen(DXF Editor)에서 Babylon.js 7.43.2 운용 경험 확보, WebView CSP 호환 검증 완료, 직교/원근 듀얼 카메라·MaterialCache·LineBatchManager 등 재활용 가능 패턴 풍부 |
| **Canvas/SVG** (matplotlib 아님) | 2D 차트(좌굴 곡선 등)는 WebView Canvas/SVG로 직접 렌더링, 3D(모드형상/소성곡면)는 Babylon.js |

## 5. 계층별 기술 매핑

### 5.1 Extension Host (TypeScript)

역할: WebView 생성, Python 프로세스 관리, 파일 I/O, VS Code API 연동

```typescript
// PythonBridge.ts - Python 프로세스 관리
import { spawn, ChildProcess } from 'child_process';

export class PythonBridge {
    private _process: ChildProcess | null = null;
    private _pendingRequests = new Map<number, (result: any) => void>();
    private _nextId = 1;

    async start(): Promise<void> {
        this._process = spawn('python', ['-m', 'cufsm_engine'], {
            cwd: this._enginePath,
            stdio: ['pipe', 'pipe', 'pipe']
        });
        // stdout에서 JSON-RPC 응답 읽기
        this._process.stdout!.on('data', (data) => {
            const response = JSON.parse(data.toString());
            const resolver = this._pendingRequests.get(response.id);
            if (resolver) { resolver(response.result); }
        });
    }

    async analyze(model: CufsmModel): Promise<CufsmResult> {
        const id = this._nextId++;
        return new Promise((resolve) => {
            this._pendingRequests.set(id, resolve);
            this._process!.stdin!.write(JSON.stringify({
                method: 'analyze',
                params: model,
                id
            }) + '\n');
        });
    }
}
```

## 7. MATLAB → WebView 시각화 매핑

### 7.1 좌굴 곡선 (thecurve3.m → Canvas)

**MATLAB:**
```matlab
semilogx(curve(:,1), curve(:,2), 'b-', 'LineWidth', 2);
xlabel('Half-wavelength'); ylabel('Load factor');
```

**WebView (Canvas):**
```javascript
function drawBucklingCurve(ctx, canvas, curveData) {
    const w = canvas.width, h = canvas.height;
    const pad = { top: 30, right: 30, bottom: 50, left: 70 };

    // semilog X 축 변환
    const xMin = Math.log10(Math.min(...curveData.map(p => p[0])));
    const xMax = Math.log10(Math.max(...curveData.map(p => p[0])));
    const yMax = Math.max(...curveData.map(p => p[1])) * 1.1;

    const toX = (val) => pad.left + (Math.log10(val) - xMin) / (xMax - xMin) * (w - pad.left - pad.right);
    const toY = (val) => h - pad.bottom - (val / yMax) * (h - pad.top - pad.bottom);

    // 곡선 그리기
    ctx.beginPath();
    ctx.strokeStyle = '#2196F3';
    ctx.lineWidth = 2;
    curveData.forEach((pt, i) => {
        i === 0 ? ctx.moveTo(toX(pt[0]), toY(pt[1]))
                : ctx.lineTo(toX(pt[0]), toY(pt[1]));
    });
    ctx.stroke();

    // 축 라벨
    ctx.fillStyle = 'var(--vscode-foreground)';
    ctx.fillText('Half-wavelength', w / 2, h - 5);
}
```

### 7.3 3D 모드형상 (dispshp2.m → Babylon.js)

stgen의 dxfRenderer.ts 패턴을 기반으로 구현.

**MATLAB:**
```matlab
surf(X, Y, Z, 'FaceColor', 'interp');
```

**WebView (Babylon.js) — stgen 패턴 재활용:**
```typescript
import { Engine } from "@babylonjs/core/Engines/engine";
import { Scene } from "@babylonjs/core/scene";
import { FreeCamera } from "@babylonjs/core/Cameras/freeCamera";
import { Camera } from "@babylonjs/core/Cameras/camera";
import { Vector3, Color4 } from "@babylonjs/core/Maths/math.vector";
import { Mesh } from "@babylonjs/core/Meshes/mesh";
import { VertexData } from "@babylonjs/core/Meshes/mesh.vertexData";
import { StandardMaterial } from "@babylonjs/core/Materials/standardMaterial";

function renderModeShape3D(canvas: HTMLCanvasElement, nodes, shapes, length, scale) {
    // stgen 패턴: Engine + Scene 초기화
    const engine = new Engine(canvas, true, { preserveDrawingBuffer: true, antialias: true });
    const scene = new Scene(engine);
    scene.useRightHandedSystem = true;  // 구조공학 좌표계
    scene.clearColor = new Color4(0.12, 0.12, 0.12, 1);

    // stgen CameraController 패턴: 직교/원근 듀얼 카메라
    const camera = new FreeCamera("cam", new Vector3(0, -length, length/2), scene);
    camera.mode = Camera.PERSPECTIVE_CAMERA;
    camera.setTarget(new Vector3(0, length/2, 0));

    // 변형된 단면을 길이방향으로 배치
    const nSteps = 20;
    const positions: number[] = [];
    const indices: number[] = [];
    const colors: number[] = [];

    for (let s = 0; s <= nSteps; s++) {
        const y = (s / nSteps) * length;
        const Ym = Math.sin(Math.PI * y / length); // S-S 경계조건
        nodes.forEach((node, i) => {
            const dx = shapes[i * 4 + 0] * Ym * scale;
            const dz = shapes[i * 4 + 1] * Ym * scale;
            positions.push(node.x + dx, y, node.z + dz);
            // 변위 크기에 따른 vertex color (컬러맵)
            const mag = Math.sqrt(dx*dx + dz*dz);
            const t = mag / (scale * 1.0); // 정규화
            colors.push(t, 0.3, 1-t, 1);   // 파랑→빨강 그라디언트
        });
    }

    // 인접 절점으로 삼각형 인덱스 생성
    const nNodes = nodes.length;
    for (let s = 0; s < nSteps; s++) {
        for (let i = 0; i < nNodes - 1; i++) {
            const a = s * nNodes + i, b = a + 1;
            const c = (s+1) * nNodes + i, d = c + 1;
            indices.push(a, b, c, b, d, c); // 두 삼각형
        }
    }

    // Babylon.js VertexData로 메시 생성
    const mesh = new Mesh("modeShape", scene);
    const vertexData = new VertexData();
    vertexData.positions = positions;
    vertexData.indices = indices;
    vertexData.colors = colors;
    vertexData.applyToMesh(mesh);

    // stgen MaterialCache 패턴
    const mat = new StandardMaterial("mat", scene);
    mat.backFaceCulling = false;
    mat.disableLighting = false;
    mesh.material = mat;

    engine.runRenderLoop(() => scene.render());
}
```

## 10. 단계별 컨버전 계획

### Phase 1: Python 해석 엔진 (3~4주)

```
목표: CLI로 좌굴 해석 실행, MATLAB 결과와 수치 검증

python/engine/
├── element.py       ← klocal.m, kglocal.m, trans.m, elemprop.m
├── assembly.py      ← assemble.m, spring_assemble.m, spring_klocal.m
├── boundary.py      ← BC_I1_5.m, constr_BCFlag.m, shapef.m
├── fsm_solver.py    ← stripmain.m
├── properties.py    ← grosprop.m, cutwp_prop2.m
└── models/data.py   ← 데이터클래스

검증: batchcufsm5.m 예제 → Python CLI → 결과 비교 (오차 < 0.1%)
```

### Phase 2: JSON-RPC 서버 + Extension 뼈대 (1~2주)

```
목표: VS Code에서 Python 엔진 호출 가능

python/server.py           ← stdin/stdout JSON-RPC
src/extension.ts           ← 커맨드 등록
src/bridge/PythonBridge.ts ← spawn + 통신
src/webview/CufsmPanel.ts  ← 빈 WebView 패널
```

### Phase 3: WebView 전처리 UI (3~4주)

```
목표: 단면 입력/편집 → 해석 실행 → 결과 수신

webview/js/panels/preprocessor.js   ← 절점/요소 테이블 편집
webview/js/panels/loading.js        ← 하중/응력 입력
webview/js/panels/boundaryCondition.js ← BC 선택
webview/js/charts/crossSection.js   ← SVG 단면 미리보기
webview/js/components/dataTable.js  ← 편집 가능 테이블
webview/js/panels/templatePicker.js ← 단면 템플릿 (C, Z, Hat 등)
```

### Phase 4: WebView 후처리 시각화 (3~4주)

```
목표: 해석 결과 시각화 (곡선, 모드형상, 분류)

webview/js/charts/bucklingCurve.js  ← Canvas semilog 곡선
webview/js/charts/modeShape2D.js    ← Canvas 2D 변형
webview/js/charts/modeShape3D.js    ← Babylon.js 3D 렌더링
webview/js/charts/classifyCurve.js  ← Canvas 모드 분류
webview/js/panels/postprocessor.js  ← 결과 탐색 UI
```

### Phase 5: cFSM + 확장 기능 (2~3주)

```
python/cfsm/                ← 모드 분류
python/fcfsm/               ← 힘 기반 구속
python/plastic/             ← 소성곡면
python/vibration/           ← 진동 해석
webview/js/panels/cfsmSettings.js
webview/js/charts/plasticSurface.js  ← Babylon.js
```

### Phase 6: 통합, 테스트, 배포 (2주)

```
├── .mat 파일 import 기능
├── 프로젝트 파일 저장/로드 (.cufsm)
├── 트리뷰 프로젝트 탐색기
├── 벤치마크 테스트 (MATLAB 대비)
└── VS Code Marketplace 배포 준비
```

## 12. 리스크 및 대응

| 리스크 | 영향 | 대응 |
|--------|------|------|
| Python 런타임 의존성 | 사용자가 Python 미설치 시 동작 불가 | Extension 활성화 시 Python 경로 자동 탐지, 미설치 시 안내 메시지 |
| `eigsh` ↔ `eigs` 수렴 차이 | 일부 케이스 결과 불일치 | shift-invert 모드 파라미터 튜닝, ARPACK 옵션 조정 |
| MATLAB 1-based → Python 0-based | 인덱스 오류 (off-by-one) | 변환 초기에 인덱싱 규칙 문서화, 단위 테스트 |
| WebView ↔ Python 대용량 데이터 전송 | 대규모 모드형상 직렬화 지연 | 결과 요약만 전송, 상세 데이터는 요청 시 lazy load |
| Babylon.js 3D 성능 | 복잡한 단면 렌더링 느림 | stgen의 LOD/배칭 패턴 적용, 절점 수 제한 |
| Canvas 인쇄/내보내기 | Canvas → 이미지 변환 품질 | SVG 병행 사용, `canvas.toDataURL()` |

## 15. stgen에서 재활용 가능한 패턴 목록

stgen(DXF Editor)은 Babylon.js 7.43.2 기반의 프로덕션급 CAD 도구로, CUFSM 개발에 직접 재활용 가능한 패턴이 풍부하다.

| stgen 소스 | CUFSM 적용 | 재활용 내용 |
|-----------|-----------|------------|
| `dxfRenderer.ts` (Engine/Scene 초기화) | 3D 모드형상 뷰어 | `Engine` 옵션, `Scene` 설정, `useRightHandedSystem`, clearColor |
| `CameraController.ts` (듀얼 카메라) | 2D 단면 뷰 + 3D 모드형상 | 직교/원근 전환, 표준 뷰(top/front/iso), 줌/팬 핸들링 |
| `MaterialCache.ts` | 응력/변위 컬러맵 | 색상별 `StandardMaterial` 캐싱, `freeze()` 최적화 |
| `LineBatchManager.ts` | 단면 요소 렌더링 | 선분 배칭으로 draw call 최소화, Float32Array 사용 |
| `EntityRenderer.ts` (메시 생성) | BufferGeometry 패턴 | `VertexData` + `applyToMesh()` 패턴 |
| `webpack.config.js` | 빌드 구성 | `maxChunks: 1`, CSP nonce, shader 인라이닝 |
| `dxfEditorProvider.ts` (WebView HTML) | CufsmPanel 뼈대 | CSP 메타태그, nonce 스크립트, postMessage 라우팅 |
| `shared/messageTypes.ts` | 메시지 프로토콜 | TypeScript 타입 기반 메시지 계약 |
| `ExportManager.ts` | 결과 이미지 내보내기 | `preserveDrawingBuffer` + Canvas→PNG/SVG 변환 |

### Babylon.js 선택 근거 (vs Three.js)

| 기준 | Three.js | Babylon.js | 판정 |
|------|----------|------------|------|
| 기존 코드 자산 | StSolar (경량 사용) | **stgen** (97개 TS 파일, 프로덕션급) | **Babylon.js** |
| WebView CSP 검증 | StSolar에서 검증 | **stgen에서 검증** (nonce, maxChunks:1) | 동등 |
| 카메라 시스템 | 직접 구현 필요 | **CameraController.ts 재활용** | **Babylon.js** |
| 재질 캐싱 | 직접 구현 필요 | **MaterialCache.ts 재활용** | **Babylon.js** |
| 선분 배칭 | 직접 구현 필요 | **LineBatchManager.ts 재활용** | **Babylon.js** |
| 번들 크기 | ~608 KB | ~3 MB (이미 stgen에서 수용) | Three.js |
| 팀 숙련도 | 중간 | **높음** (stgen 개발 경험) | **Babylon.js** |
| webpack 설정 | 새로 구성 | **stgen 설정 복사** | **Babylon.js** |
