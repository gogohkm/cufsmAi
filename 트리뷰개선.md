# CUFSM 트리뷰 개선 작업 결과

## 개요

VS Code 사이드바의 프로젝트 탐색기(트리뷰)를 현재 웹뷰의 6개 탭 구조와
입력/결과 데이터를 반영하도록 전면 개편하였다.

## 변경 파일

| 파일 | 변경 내용 |
|------|-----------|
| `src/webview/ProjectExplorerProvider.ts` | 트리 구조 전면 재작성 (6탭, 30+ 항목) |
| `src/webview/CufsmPanel.ts` | `_updateTreeView(extra)` 확장, `treeUpdate` 메시지 핸들러 추가 |
| `webview/js/app.js` | `sendTreeUpdate()` 함수 추가, `handleShowSection` 확장 (focus 네비게이션) |

## 주요 기능

### 1. 클릭 → 해당 입력칸 포커스 이동

트리 항목을 클릭하면 해당 탭으로 전환 후, 관련 입력 요소로 스크롤 & 포커스.

| 트리 항목 | 이동 탭 | 포커스 대상 |
|-----------|---------|-------------|
| 단면 템플릿 | 전처리 | `select-template` |
| 치수 | 전처리 | `tpl-H` |
| 재료 | 설계 | `design-fy` |
| 부재 유형 | 설계 | `select-member-type` |
| 스팬 구성 | 설계 | `select-span-type` |
| 간격 | 설계 | `config-spacing` |
| 비지지 길이 | 설계 | `design-Lb` |
| 각 하중 (D/Lr/S/L/W) | 설계 | 해당 하중 입력 |
| 최대 Mu | 설계 | `design-Mx` |
| 최대 Vu | 설계 | `design-V` |
| 이용률 DCR | 설계 | `design-summary` |
| 실패/주의/통과 항목 | 검증 | `validation-container` |

### 2. 실시간 데이터 갱신

다음 이벤트 발생 시 트리뷰가 자동 갱신됨:

| 이벤트 | 갱신 데이터 |
|--------|-------------|
| 단면 로드/생성 | 절점수, 요소수, 단면 유형 |
| 단면 성질 계산 | A, Ixx, Sx, rx, rz |
| FSM 해석 완료 | hasResults, DSM 값 (Pcrl, Mcrl 등) |
| 하중 분석 완료 | 지배 조합, Mu, Vu, 처짐 |
| 설계 검토 완료 | 지배 모드, φMn/φPn, 이용률, OK/NG |
| 검증 실행 | 통과/주의/실패 개수 |

### 3. 자동 실행 기능

| 트리 항목 | 동작 |
|-----------|------|
| `해석 실행` (▶) | 클릭 시 FSM 좌굴 해석 자동 시작 |
| `검증 실행` (▶) | 클릭 시 설계 검증 자동 실행 |

### 4. 조건부 표시

- DSM 값 (Pcrl/Mcrl 등): FSM 해석 완료 후에만 표시
- 하중 분석 결과: 하중 분석 실행 후에만 표시
- 설계 결과: 설계 검토 실행 후에만 표시
- 개별 하중: 값이 입력된 하중만 표시 (0인 하중은 숨김)

## 데이터 흐름

```
[WebView app.js]
    │
    │  sendTreeUpdate()
    │  (모델/해석/설계 상태 변경 시 호출)
    │
    ▼
{ command: 'treeUpdate', data: { ... } }
    │
    ▼
[CufsmPanel.ts]
    │
    │  case 'treeUpdate':
    │    this._updateTreeView(message.data)
    │
    ▼
[ProjectExplorerProvider.ts]
    │
    │  updateProjectData(summary)
    │  → _onDidChangeTreeData.fire()
    │
    ▼
[VS Code 사이드바 트리뷰 갱신]
```

```
[사용자가 트리 항목 클릭]
    │
    ▼
[extension.ts]
    │  treeView.onDidChangeSelection
    │  → CufsmPanel.showSection(sectionId)
    │
    ▼
[CufsmPanel.ts]
    │  _postMessage('showSection', { sectionId })
    │
    ▼
[WebView app.js]
    │  handleShowSection(sectionId)
    │  → switchTab(tab)
    │  → element.scrollIntoView + focus()
    │
    ▼
[해당 탭/입력칸으로 이동]
```

## ProjectSummary 인터페이스 (확장)

기존 6개 필드에서 30+ 필드로 확장:

```typescript
interface ProjectSummary {
    // 전처리
    name, nnodes, nelems, sectionType,
    H, B, D, t,
    steelGrade, Fy, Fu,
    // 해석
    BC, nlengths, hasResults,
    // 단면 성질
    A, Ixx, Sx, rx, rz,
    // DSM
    Pcrl, Pcrd, Mcrl, Mcrd,
    // 설계
    memberType, spanType, designMethod,
    spanLength, spacing,
    loadD, loadLr, loadS, loadW, loadL,
    // 설계 결과
    hasDesignResult, designMn, designPn,
    controllingMode, utilization, passOrFail,
    // 하중 분석
    hasLoadAnalysis, gravityCombo, maxMu, maxVu,
    maxDeflection, deflectionRatio,
    // 검증
    validationPass, validationWarn, validationFail,
}
```

`updateProjectData()`는 `Partial<ProjectSummary>`를 받아 기존 값과 병합하므로,
일부 필드만 전달해도 나머지는 유지됨.
