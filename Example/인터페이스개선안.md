### 2.1 워크플로우 시각화 — 스텝 인디케이터 (★★★)

**문제:** Analyze Loads → Run Design의 2단계 흐름이 시각적으로 분리되지 않아, 사용자가 어떤 단계에 있는지 파악하기 어려움.

**개선안:** 좌측 패널 상단에 스텝 인디케이터 추가.

```
┌────────────────────────────────────────┐
│  ① Inputs ─── ② Loads ─── ③ Design    │  ← 스텝 인디케이터
│     ●━━━━━━━━━━○━━━━━━━━━━○            │
│  (현재 단계 강조, 완료 단계 체크마크)    │
└────────────────────────────────────────┘
```

**단계 정의:**

| 단계 | 이름 | 활성 조건 | 포함 섹션 |
|------|------|----------|----------|
| ① Inputs | 입력 정의 | 항상 | Material, Method, Member Type, Configuration, Loads, Deck |
| ② Loads | 하중 해석 | Calculator 모드일 때 | [Analyze Loads] → 결과 표시 |
| ③ Design | 설계 검토 | Inputs 완료 후 | Unbraced Lengths, Required Loads, [Run Design] → 결과 표시 |

**Direct Input 모드:** ① → ③ (② 건너뜀)
**Calculator 모드:** ① → ② → ③ (② 완료 시 자동으로 ③의 입력 필드 채움)

**구현:**


### 2.2 좌측 패널 구조 개편 — 접이식 개선 (★★★)

**문제:**
1. 접이식 토글이 `▸` 문자 하나에 의존 — 클릭 가능 여부 불분명
2. 펼침/접힘 상태 표시 없음 (`▸`↔`▾` 전환 미구현)
3. 모든 섹션이 초기에 펼쳐져 있어 스크롤 과다

**개선안:**

#### A. 토글 아이콘 상태 전환

```html
<h3 class="collapsible" data-expanded="true">
  <span class="collapse-icon">▾</span> Member Configuration
</h3>
<div class="collapsible-content">
  <!-- 내용 -->
</div>
```

```javascript
document.querySelectorAll('.collapsible').forEach(h3 => {
  h3.addEventListener('click', () => {
    const content = h3.nextElementSibling;
    const icon = h3.querySelector('.collapse-icon');
    const expanded = h3.dataset.expanded === 'true';
    
    content.style.display = expanded ? 'none' : 'block';
    icon.textContent = expanded ? '▸' : '▾';
    h3.dataset.expanded = String(!expanded);
  });
});
```


#### B. 초기 상태: 관련 없는 섹션 자동 접기

| 모드 | 초기 펼침 | 초기 접힘 |
|------|----------|----------|
| Direct Input (General) | Material, Method, Member Type, Lengths, Loads | Web Crippling |
| Calculator (Application) | Material, Method, Member Type, Configuration | Loads, Deck, Lengths |

**이유:** 사용자가 순서대로 채워나가도록 하되, 첫 화면에서 보이는 항목을 최소화하여 인지 부하를 줄임.

입력이 유효하면 섹션 h3 우측에 작은 체크 아이콘 표시:

```
▾ Service Loads ✓     ← D가 0보다 큰 값
▸ Deck & Bracing      ← 아직 미확인
```

### 2.3 결과 패널 가독성 개선 (★★★)

**문제:** Step-by-Step Calculation이 단색 monospace 텍스트로 되어 있어 핵심 값을 빠르게 찾기 어려움.

#### A. 계산 단계를 카드 형태로 변경

현재:
```
1. Yield (Py = Ag×Fy)  Py = 37.35 kips
2. Global (§E2)  Fcre = 30.5 ksi, Pne = 21.3 kips  ← controlling
3. Local (§E3.2)  λl = 0.787, Pnl = 20.96 kips
```

개선:
```
┌─ 1. Yield ─────────────────────────────┐
│  Py = Ag × Fy                          │
│  Py = 0.679 × 55.0 = 37.35 kips       │
└────────────────────────────────────────┘

┌─ 2. Global Buckling (§E2) ────── ◀ ───┐  ← 지배 모드 강조
│  Fcre = 30.5 ksi                       │
│  λc = √(Fy/Fcre) = 1.34               │
│  Fn = 0.658^(1.34²) × 55 = 31.4 ksi  │
│  Pne = Ag × Fn = 21.3 kips            │
└────────────────────────────────────────┘

┌─ 3. Local Buckling (§E3.2) ───────────┐
│  λℓ = √(Pne/Pcrl) = 0.787             │
│  Pnl = 20.96 kips                      │
└────────────────────────────────────────┘
```

### 2.5 입력 검증 및 피드백 시스템 (★★)

**문제:** 잘못된 입력(음수 Fy, 0인 단면 길이 등)에 대한 피드백이 전혀 없음.

#### A. 실시간 입력 검증

```javascript
const validationRules = {
  'design-fy':    { min: 1, max: 100, label: 'Fy' },
  'design-fu':    { min: 1, max: 120, label: 'Fu' },
  'design-KxLx':  { min: 0, max: 10000, label: 'KxLx' },
  'design-KyLy':  { min: 0, max: 10000, label: 'KyLy' },
  'design-Cb':    { min: 1.0, max: 3.0, label: 'Cb' },
  'config-span':  { min: 0.5, max: 100, label: 'Span' },
};

function validateInput(id) {
  const el = document.getElementById(id);
  const rule = validationRules[id];
  if (!el || !rule) return true;
  
  const val = parseFloat(el.value);
  const valid = !isNaN(val) && val >= rule.min && val <= rule.max;
  
  el.classList.toggle('input-invalid', !valid);
  
  // 인라인 에러 메시지
  let errEl = el.nextElementSibling;
  if (!errEl || !errEl.classList.contains('input-error')) {
    errEl = document.createElement('span');
    errEl.className = 'input-error';
    el.parentNode.insertBefore(errEl, el.nextSibling);
  }
  errEl.textContent = valid ? '' : `${rule.label}: ${rule.min}~${rule.max}`;
  errEl.style.display = valid ? 'none' : 'inline';
  
  return valid;
}
```

### 2.7 단위 시스템 토글 (★)

**문제:** 모든 입력이 US Customary (ksi, in, kips) 고정.

**개선안:** 향후 확장을 위한 단위 토글 드롭다운 (SI/US).

```
┌─ Material ──────────────────────┐
│  Steel Grade: [A653 Gr.50 ▼]   │
│  Units: [US Customary ▼]       │  ← 신규
│  Fy: [50] ksi   Fu: [65] ksi   │
└─────────────────────────────────┘
```

### 6.1 현재 문제: 정보의 분산과 중복

동일한 파라미터가 여러 탭에 흩어져 있어 입력 혼란과 동기화 버그의 원인이 됨.

| 파라미터 | 전처리 탭 | 해석 탭 | 설계 탭 | 문제 |
|---------|----------|---------|---------|------|
| **E, v, G** | ✓ (Material) | — | — | OK (1곳) |
| **Fy** | — | `input-fy-load` | `design-fy` | **중복** |
| **Fu** | — | — | `design-fu` | OK (1곳) |
| **강종** | — | — | `select-steel-grade` | OK (1곳) |
| **BC (FSM)** | — | `select-bc` | — | OK (해석 파라미터) |
| **경간/지점** | — | — | 퍼린 Configuration | OK (설계 파라미터) |

### 6.2 개선 원칙

**"앞 탭에서 결정된 것은 뒤 탭에서 중복하지 않는다"**

```
전처리 탭               해석 탭                설계 탭
──────────────        ──────────             ─────────────────
① 단면 치수             ④ 반파장 범위          ⑧ 부재 유형
② 재료 (E, v → G 자동)  ⑤ 반파장 개수          ⑨ 소요강도 (Mu, Pu)
③ 강종/Fy/Fu ← [이동]   ⑥ 고유치 수            ⑩ 비지지 길이
                       ⑦ BC (S-S, C-F...)     ⑪ 설계법 (ASD/LRFD)
                          Load Case           ⑫ 퍼린 경간/지점/Lap
                          (Fy는 전처리에서 참조)
```

### 6.3 변경 사항

#### A. Fy/Fu/강종을 전처리 탭으로 이동

- 전처리 Material 섹션에 강종 선택 + Fy + Fu 추가
- 해석 탭의 `input-fy-load` 제거 → 전처리 Fy 자동 참조
- 설계 탭의 `design-fy`, `design-fu` → 읽기 전용 표시 (전처리 값)
- 후처리 탭의 `input-fy` 제거 → 전처리 Fy 자동 참조

#### B. 해석 탭의 Load Case 응력 설정

현재: Load Case 변경 시 `input-fy-load`에서 Fy를 읽어 응력 설정
변경: `model.prop` 또는 전처리 Fy를 직접 참조

```javascript
// 변경 전: 해석 탭의 별도 Fy 입력 사용
const fyLoad = fromDisplay(getNum('input-fy-load', 52.94), 'stress');

// 변경 후: 전처리 탭의 Fy를 참조
const fyLoad = fromDisplay(getNum('input-fy', 52.94), 'stress');
```

#### C. Fy 변경 시 재해석 알림

전처리에서 Fy를 변경하면 기존 해석 결과가 무효화되므로, 재해석 필요 알림 표시:

```javascript
fyInput.addEventListener('input', () => {
    if (analysisResult) {
        showWarning('Fy 변경됨 — 해석을 다시 실행하세요');
    }
});
```

### 6.4 FSM BC vs 부재 지점조건

**해석 탭의 BC (S-S, C-C 등)와 설계 탭의 경간/지점 조건은 다른 개념:**

| | FSM BC (해석 탭) | 부재 지점 (설계 탭) |
|---|---|---|
| **의미** | 반파장 끝단의 종방향 구속 | 부재 전체의 지지 조건 |
| **영향** | 좌굴 모드 형상 | 모멘트 분포, 비지지 길이 |
| **예시** | S-S (simply supported) | cont-3 (3경간 연속) |

**AISI 예제(II-1B/1C) 검증 결과:**
- 외경간(힌지-연속)과 내경간(연속-연속)의 차이는 FSM BC가 아닌 **Cb와 Lb**로 구분
- 모든 경간에서 FSM은 **S-S로 해석** → Mcrl, Mcrd 산출
- LTB(Mcre)는 설계 단계에서 Cb 보정으로 처리

**좌굴 모드별 FSM BC 영향:**

| 좌굴 모드 | 반파장 | FSM BC 영향 | 처리 방법 |
|----------|--------|-----------|----------|
| 국부좌굴 (Mcrl) | 수 인치 | 없음 | S-S로 충분 |
| 뒤틀림좌굴 (Mcrd) | 수십 인치 | 미미 | S-S (약간 보수적) |
| 전체좌굴 (Mcre) | 비지지 길이 | 있으나 | **Cb, Lb로 설계 단계 보정** |

**예외:** 캔틸레버 부재 → FSM BC를 `C-F`로 변경 필요

### 6.5 구현 후 정보 흐름

```
강종 변경(전처리) → ⚠️ 재해석 필요 알림
                  → 해석 실행 (Fy 자동 참조)
                  → DSM 추출 (_getAnalysisFy)
                  → 설계 (Fy 전처리에서 읽기)
```

**syncFy() 불필요** — Fy가 전처리에 단일 소스로 존재하므로 동기화 코드 제거 가능
