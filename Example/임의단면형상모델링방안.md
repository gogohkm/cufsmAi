# 임의 단면 형상 모델링 방안

## CFS(냉간성형강) 임의 단면을 정확히 모델링하기 위한 체계적 방법론

작성일: 2026-04-05 (3차 갱신)

---

## 1. 근본 문제

CFS 단면은 **한 장의 평판을 순차적으로 접은 것**이다. 2D 도면은 이 접힌 결과의 **투영**이므로, 같은 도면에서 여러 해석이 가능하다. 좌표를 직접 나열하는 방식은 **매번 처음부터 해석해야 하므로** 실수가 반복된다.

**해결 방향**: 좌표를 직접 나열하지 말고, **구조 요소(building block)를 조합**하여 자동 생성한다.

---

## 2. CFS 단면의 구조 요소 분해

모든 CFS 열린 단면은 아래 **5가지 기본 요소**의 조합으로 분해할 수 있다:

### 2.1 기본 요소 라이브러리

| 요소 | 세그먼트 수 | 경로 패턴 | 예시 |
|------|-----------|----------|------|
| **Simple Lip** | 1 | 직선 (플랜지에서 안쪽으로) | C형 립, Z형 립 |
| **L-Flange** | 1 | 직선 (웹에서 수평으로) | 일반 C/Z 플랜지 |
| **Track Flange (∪/∩)** | 3 | 직선→직선→**되돌아감** | 시그마 플랜지, Track 단면 |
| **Straight Web** | 1 | 수직 직선 | 모든 단면의 웹 |
| **Step Stiffener** | 3 | 수평→수직→수평(복귀) | 시그마 웹 스티프너 |

### 2.2 요소의 방향 규칙

각 요소는 **어느 방향으로 뻗는지**가 핵심:

```
Simple Lip:     플랜지 끝에서 "안쪽"(단면 내부 방향)으로 꺾임
L-Flange:       웹에서 "바깥쪽"으로 수평 연장
Track Flange:   플랜지→립 방향으로 갔다가 "되돌아옴" (U-turn)
Straight Web:   위에서 아래로 (또는 아래에서 위로)
Step Stiffener: 웹에서 수평 돌출 후 복귀
```

### 2.3 연결 규칙

```
[Lip] ──→ [Flange] ──→ [Web] ──→ [Stiffener] ──→ [Web] ──→ [Flange] ──→ [Lip]
```

**연결 시 방향 전환**:
- Lip→Flange: 90° 꺾임 (수직→수평 또는 수평→수직)
- Flange→Web: 90° 꺾임
- Web→Stiffener: 90° 꺾임 (수평 돌출)
- **Track Flange 내부**: 180° 되돌아감 (U-turn)

## 3. 단면 모델링 프로토콜 (Step-by-Step)

### Step 1: 도면에서 구조 요소 식별

도면을 보고 아래 질문에 답한다:

```
Q1. 이 단면은 어떤 글자/기호 모양인가?
    → C, Z, Σ(시그마), Ω(오메가), Hat(∩), Rack 등

Q2. 구조 요소 목록을 작성하라:
    예) Σ = [상부 Track 플랜지] + [상부 웹] + [Step 스티프너] + [하부 웹] + [하부 Track 플랜지]

Q3. 각 요소의 치수는?
    예) Track 플랜지: B=2.25, D=0.875, D_lip=0.5

Q4. 대칭인가? 어떤 유형?
    → 거울대칭 (상하 같은 방향) / 점대칭 (상하 반대 방향) / 비대칭

Q5. 각 요소는 어느 방향으로 뻗는가?
    → 플랜지: 웹에서 우측으로
    → 스티프너: 웹에서 좌측으로 (또는 우측으로)
    → 립: 플랜지에서 안쪽으로
```

### Step 2: 기준점(앵커) 설정

좌표 계산 전에 **기준점**을 정한다:

```
규칙:
- 웹의 x좌표 = 0 (또는 플랜지가 뻗는 반대쪽)
- 하단 = y=0, 상단 = y=H
- 모든 좌표는 이 기준에서 계산
```

### Step 3: 요소별 좌표 계산

**각 요소를 독립적으로 계산**한 후 연결한다. 전체를 한꺼번에 계산하지 않는다.

```
예: 시그마 상부 Track 플랜지 (B=2.25, D=0.875, D_lip=0.5)
    웹 접합점이 (0, 7.125)일 때:
    
    경로: 립 끝 → 위(D) → 우(B) → 아래(D, 되돌아감) → 웹 접합
    
    P0 = (B+D_lip, H-D) = (2.75, 7.125)    ← 립 끝 (자유단)
    P1 = (B+D_lip, H)   = (2.75, 8.0)      ← 립 상단
    P2 = (0, H)          = (0, 8.0)          ← 플랜지 상단 좌측
    P3 = (0, H-D)        = (0, 7.125)        ← 웹 접합 (되돌아감!)
```

### Step 4: 대칭 적용

거울대칭인 경우, **상부 요소의 좌표를 y축 기준으로 반전**하여 하부 생성:

```python
# 상부 플랜지 좌표가 [(2.75, 7.125), (2.75, 8), (0, 8), (0, 7.125)]일 때
# y_center = H/2 = 4.0
# 하부 플랜지 = [(2.75, 0.875), (2.75, 0), (0, 0), (0, 0.875)]
#            = 각 (x, H - y)로 변환하고 순서 반전

# 또는 단순히: 하부 y좌표 = H - 상부 y좌표
```

### Step 5: 좌표 조합 및 경로 연결

요소별 좌표를 **경로 순서대로** 연결한다. 중복 점 제거.

### Step 6: set_custom_section 호출 및 시각 확인

```
set_custom_section(outer_corners=..., t=..., R_inner=...)
→ get_section_preview() → Read → 도면과 비교
→ 불일치 시 해당 요소만 수정 (전체 재계산 아님!)
```

### Step 7: 검증

```
get_section_properties()
→ θp ≈ 0° (대칭 단면)
→ A ≈ 예상값 (±10%)
→ Ixx, Sx 합리적 범위
```

## 4. 방향 판별 규칙 (가장 실수가 많은 부분)

### 4.1 립의 방향

```
규칙: 립은 항상 "안쪽"(단면 내부)으로 꺾인다.

C형: 립이 웹 쪽으로 → 플랜지가 우측이면 립은 좌측으로
Z형: 상부 립과 하부 립이 반대 방향
시그마: C형과 동일 → 립이 단면 내부로

판별법: 립이 꺾이는 방향이 단면의 "무게중심" 쪽이면 맞다.
```

### 4.2 Track 플랜지의 되돌아감 방향

```
규칙: Track 플랜지는 "개구부"가 단면 내부를 향한다.

시그마 상부: 개구부가 아래 → 경로가 위→좌→아래(되돌아감)
시그마 하부: 개구부가 위 → 경로가 아래→우→위(되돌아감)

판별법: Track의 개구부가 웹 쪽을 향하면 맞다.
```

### 4.3 스티프너의 돌출 방향

```
규칙: 스티프너는 웹에서 "플랜지 반대쪽"으로 돌출하는 경우가 많다.
      하지만 도면을 확인해야 한다.

시그마: 플랜지가 우측 → 스티프너가 좌측(또는 우측, 도면 확인)
```

### 4.4 outer_side 판별

```
규칙: 경로의 P0→P1 방향으로 진행할 때, 판의 바깥면이 어느 쪽인지.

실용적 방법:
1. P0(립 끝)에서 P1(립-플랜지 코너) 방향으로 진행
2. 이때 판의 "바깥면"(외측)이 좌측이면 'left', 우측이면 'right'
3. 확신이 없으면 양쪽 다 시도하고 preview로 확인
```

## 5. 수정 시 안전 규칙

### 5.1 좋은 결과 보존

```
규칙: Mcrl/My가 AISI 대비 ±5% 이내면 "좋은 결과"로 간주.
      수정 전 반드시 현재 좌표를 기록.
```

### 5.2 최소 범위 수정

```
규칙: 특정 요소만 수정할 때, 해당 요소의 좌표(2~4점)만 변경.
      나머지 요소의 좌표는 절대 건드리지 않는다.

예: "상부 립 방향이 다르다" → P0, P1만 수정 (나머지 P2~P11 유지)
```

### 5.3 수정 후 검증

```
규칙: 수정 후 반드시:
1. get_section_preview() → 형상 확인
2. get_section_properties() → θp 확인
3. run_analysis() → Mcrl/My 확인
4. 이전보다 악화되면 → 즉시 롤백
```

### 5.4 롤백 프로토콜

```
if (수정 후 Mcrl/My 오차 > 이전 오차):
    → 이전 좌표로 복원
    → "미세 형상 차이는 허용" 판단
    → 수정 중단
```

## 6. 실패 사례 분석

### 6.1 시그마 단면 12회 시도 실패 원인

| 시도 | 핵심 실수 | 교훈 |
|------|----------|------|
| 1~3 | V자 스티프너로 가정 | 도면에서 요소 유형을 정확히 식별해야 |
| 4~5 | 상하 비대칭으로 해석 | "대칭"의 유형을 확인해야 (거울/점) |
| 6~8 | 플랜지를 L형으로 모델링 | Track형(3변) vs L형(2변) 구분 필요 |
| 9 | 경로 시작점 오류 | 자유단(립 끝)에서 시작해야 |
| 10~11 | 스티프너 치수 혼동 | 치수가 "무엇의" 치수인지 확인 |
| **12** | **성공** | 구조 요소 분해 + 되돌아감 인식 |
| 13 | 좋은 결과를 수정하다 망침 | 수정은 최소 범위, 악화 시 롤백 |

| 14 | 립 방향 비대칭 미감지 | **프리뷰의 z축이 반전**되어 상하 판단 불가 |

### 6.2 z축 반전 문제 (추가 발견, 시도 14)

**문제**: Cross Section Preview의 z축 방향이 Mode Shape와 **반대**였음.
- SVG 좌표계: y축이 **아래**로 증가
- 구조공학 z축: **위**로 증가
- 결과: 프리뷰에서 단면이 **상하 반전**되어 표시 → 립 방향 판단 불가

**수정**: `renderSectionSVG()`에서 z좌표를 `-n[2]`로 반전, viewBox도 대응 수정.

**교훈**: **프리뷰가 틀리면 모든 판단이 틀린다.** 좌표계 방향은 모든 시각적 검증의 전제 조건이다.

### 6.3 거울대칭에서의 립 방향 규칙 (추가, 시도 14)

거울대칭 단면에서 **상부와 하부의 립 방향**:
- 상부 립: 플랜지 끝에서 **안쪽**(단면 중심 방향)으로 꺾임
- 하부 립: 플랜지 끝에서 **안쪽**(단면 중심 방향)으로 꺾임
- 거울대칭이므로: 상부 립이 "아래+안쪽"이면 하부 립은 "위+안쪽"

**이전 실수**: 상부 립을 `left`(좌측으로), 하부 립도 `left`로 설정 → 하부는 맞지만 상부는 틀림.
상부 립은 `right`(우측=안쪽)여야 했음.

**규칙 추가**: 립의 방향은 항상 **단면의 무게중심 쪽**을 향한다.

### 6.4 핵심 깨달음

1. **좌표를 직접 나열하는 것은 위험하다.** 구조 요소를 먼저 분해하고, 요소별로 좌표를 계산해야 한다.
2. **"되돌아감"은 Track 플랜지의 본질이다.** 시그마의 플랜지는 단순 L이 아니라 ∪/∩형이다.
3. **시각 확인 없이는 모델링이 불가능하다.** `get_section_preview`는 필수.
4. **프리뷰의 좌표계가 틀리면 모든 판단이 틀린다.** z축 방향은 반드시 Mode Shape와 일치해야 한다.
5. **립은 항상 안쪽(단면 중심)을 향한다.** 거울대칭에서 상하 립의 방향이 다를 수 있다.
4. **좋은 결과가 나오면 멈춰야 한다.** 미세 차이를 수정하려다 전체를 망가뜨리는 패턴을 경계.

## 8. 구현 완료된 코드 개선

### 8.1 SectionBuilder (구현 완료)

구조 요소(building block) 조합으로 외측 꼭짓점 자동 생성:

```python
b = SectionBuilder()
b.set_expected_center(0, 4)            # 립 방향 자동 판별용 중심 힌트
b.start(1.75, 7.125)                   # 자유단
b.add_lip(0.5, 'right')               # 립
b.add_web(0.875, 'up')                # 수직벽 (되돌아감)
b.add_flange(2.25, 'left')            # 플랜지
b.add_web(2.25, 'down')               # 웹
b.add_stiffener(0.5, 1.75, 'right')   # 스티프너 (3점 자동)
b.add_web(2.25, 'down')               # 웹
b.add_flange(2.25, 'right')           # 하부 플랜지
b.add_web(0.875, 'up')                # 수직벽 (되돌아감)
b.add_lip(0.5, 'left')                # 하부 립
corners = b.build()                    # 12점 외측 꼭짓점
```

MCP: `build_section` 도구

### 8.2 lip_inward + expected_center (구현 완료)

립 방향 자동 판별. 경로 초기에 중심 추정이 부정확한 문제를 `set_expected_center()`로 해결:

```python
b.set_expected_center(0, 4)   # H=8 단면의 대략 중심
b.add_lip_inward(0.5)         # 자동으로 단면 중심 방향
```

**주의**: `expected_center` 없이 경로 시작점에서 `lip_inward` 사용 시 방향이 틀릴 수 있음.
비대칭 단면에서는 `add_lip(length, direction)`으로 수동 지정이 더 안전.

### 8.3 mirror_y (구현 완료, 제한적 사용)

거울대칭 단면에서 하부 자동 생성:

```python
b.mirror_y(y_center=4.0)  # 하부 = 상부의 y=4 기준 반사
```

**주의**: 비대칭 단면(Z형, Rack 등)에서는 사용 불가. 거울대칭 확인 후에만 사용.

### 8.4 validate_section 립 방향 검증 (구현 완료)

자동으로 립의 끝점이 단면 중심을 향하는지 검증:

