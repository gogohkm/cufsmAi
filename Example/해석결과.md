### 5.2 잔여 이슈

#### NOTE*: 8ZS2.25x059 뒤틀림 (II-2B, III-7B)
- 깊은 웹(H/B=3.56) + 긴 립(D/B=0.40) Z-section에서 뒤틀림 불일치
- **근본 원인 규명 (2026-04-04)**:
  1. **Python FSM 엔진은 정확함**: MATLAB zwlip_Mxr.mat 파일의 node/elem을 Python 솔버에 그대로 입력하면 MATLAB 결과와 0.00% 오차로 **완벽 일치** (50개 길이 전체)
  2. **AISI 기대값(0.77)의 출처**: MATLAB .mat 참조 파일은 qlip=90이 아닌 **경사 립(약 130도)**을 가진 다른 단면 형상. 플랜지 폭(B=1.95)과 립 길이도 표준 8ZS2.25x059(B=2.25)과 다름
  3. **Z-section 뒤틀림 곡선 특성**: Z-section 휨 시 시그니처 곡선의 뒤틀림 범위(L=10~30) 최소값은 **인장 플랜지** 뒤틀림 모드에 지배됨 (모드 형상 분석으로 확인). 실제 압축 플랜지 뒤틀림은 cFSM 모드 분해로만 분리 가능
  4. **cFSM 뒤틀림 모드 분석**: D-only 구속 시 최소값 1.21, G+D 구속 시 1.20 (비구속 1.14보다 높음). 이는 시그니처 곡선 최소값이 뒤틀림이 아닌 G/D 결합 모드임을 확인
- cFSM Python 포팅 완료 (2880줄), classify_modes MCP 동작 확인

#### NOTE**: II-6B 약축 휨 — 해결 (2026-04-04)

- **원인 규명 및 해결 완료**: Mcrl/My = 1.417 (AISI 1.42, 오차 -0.2%)
- 원인 1: `lippedc`(D=0.001)로 트랙 근사 시 플랜지 길이 공식 차이 (b=B-t vs b=B-t/2) → `track` 전용 템플릿 추가로 해결
- 원인 2: Mzz=+1은 플랜지 압축, 케이블 트레이(II-6B)는 웹 압축 → Mzz=-1 적용으로 해결
